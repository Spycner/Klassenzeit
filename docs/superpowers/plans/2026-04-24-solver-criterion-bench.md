# Solver Criterion Bench Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire `mise run bench` to a criterion benchmark that solves the Grundschule fixture 200 times, produce a committed `solver/solver-core/benches/BASELINE.md` with p1/p50/p99/placements-per-second, and ship `mise run bench:record` as the refresh path.

**Architecture:** Add `criterion = "0.5"` as a workspace dev-dependency. Create one `[[bench]] harness = false` target in `solver-core`, `benches/solver_grundschule.rs`, that builds a local Grundschule `Problem` (duplicate of the test fixture with a TODO pointer), benches `solve(&problem)` via `Criterion::iter_custom` collecting per-iteration `Duration`s, and prints a fenced `---SOLVER-BENCH-BASELINE---` block to stderr. A bash helper `scripts/record_solver_bench.sh` re-runs the bench, parses the fenced block, captures host info, and renders `solver/solver-core/benches/BASELINE.md`. `mise run bench:record` shells out to the script.

**Tech Stack:** Rust 1.85 (workspace toolchain), `criterion` 0.5 (workspace dev-dep), bash, `mise` task runner.

//! Criterion bench for the MVP solver over a Grundschule-shaped fixture.
//!
//! The fixture builder below is a deliberate duplicate of
//! `solver-core/tests/grundschule_smoke.rs::grundschule()`. Sprint item 6
//! on `docs/superpowers/OPEN_THINGS.md` (benchmark-fixture matrix) will
//! consolidate both onto a shared builder with pre-assigned teacher ids;
//! until then this copy is the bench source of truth.
//!
//! Output contract: after `group.finish()` we print a tab-separated block
//! fenced by `---SOLVER-BENCH-BASELINE---` / `---END---` to stderr.
//! `scripts/record_solver_bench.sh` depends on those markers, not on
//! criterion's default output format.

use std::time::Duration;

use criterion::{criterion_group, criterion_main, BenchmarkGroup, Criterion, SamplingMode};

/// Return `(p1, p50, p99)` as `Duration` values by sorting `samples` in place
/// and indexing at floor(0.01 * N), floor(0.50 * N), floor(0.99 * N).
///
/// Panics if `samples` is empty; the bench never calls this with zero samples.
fn compute_percentiles(samples: &mut [Duration]) -> (Duration, Duration, Duration) {
    assert!(!samples.is_empty(), "compute_percentiles needs >= 1 sample");
    samples.sort_unstable();
    let n = samples.len();
    let idx = |p: f64| -> usize {
        let raw = (p * n as f64).floor() as usize;
        raw.min(n - 1)
    };
    (samples[idx(0.01)], samples[idx(0.50)], samples[idx(0.99)])
}

fn bench_grundschule(_c: &mut Criterion) {
    panic!("bench_grundschule not yet implemented (Task 3 turns this green)");
}

criterion_group!(benches, bench_grundschule);
criterion_main!(benches);

#[cfg(test)]
mod tests {
    use super::compute_percentiles;
    use std::time::Duration;

    fn us(micros: u64) -> Duration {
        Duration::from_micros(micros)
    }

    #[test]
    fn single_sample_has_equal_percentiles() {
        let mut s = [us(100)];
        let (p1, p50, p99) = compute_percentiles(&mut s);
        assert_eq!(p1, us(100));
        assert_eq!(p50, us(100));
        assert_eq!(p99, us(100));
    }

    #[test]
    fn three_sorted_samples_pick_first_second_third() {
        // N=3, 0.01*3 -> 0, 0.50*3 -> 1, 0.99*3 -> 2.
        let mut s = [us(10), us(20), us(30)];
        let (p1, p50, p99) = compute_percentiles(&mut s);
        assert_eq!(p1, us(10));
        assert_eq!(p50, us(20));
        assert_eq!(p99, us(30));
    }

    #[test]
    fn unsorted_input_is_sorted_in_place() {
        let mut s = [us(30), us(10), us(20)];
        let _ = compute_percentiles(&mut s);
        assert_eq!(s, [us(10), us(20), us(30)]);
    }

    #[test]
    fn two_hundred_samples_pick_expected_positions() {
        // Ascending 1..=200 microseconds; floor(0.01 * 200) = 2 -> samples[2] = 3.
        // floor(0.50 * 200) = 100 -> samples[100] = 101.
        // floor(0.99 * 200) = 198 -> samples[198] = 199.
        let mut s: Vec<Duration> = (1..=200).map(us).collect();
        let (p1, p50, p99) = compute_percentiles(&mut s);
        assert_eq!(p1, us(3));
        assert_eq!(p50, us(101));
        assert_eq!(p99, us(199));
    }

    #[test]
    fn indices_are_clamped_to_last_element() {
        // With N=1, floor(0.99 * 1) = 0 regardless. Doubly-make-sure the
        // clamp catches any corner case the floor doesn't.
        let mut s = [us(42)];
        let (_, _, p99) = compute_percentiles(&mut s);
        assert_eq!(p99, us(42));
    }
}

```rust
use std::collections::HashSet;
use std::time::{Duration, Instant};

use criterion::{criterion_group, criterion_main, Criterion, SamplingMode};
use solver_core::{
    ids::{LessonId, RoomId, SchoolClassId, SubjectId, TeacherId, TimeBlockId},
    solve,
    types::{
        Lesson, Problem, Room, RoomSubjectSuitability, SchoolClass, Subject, Teacher,
        TeacherQualification, TimeBlock,
    },
};
use uuid::Uuid;

const BENCH_SAMPLE_COUNT: usize = 200;

fn bench_uuid(n: u8) -> Uuid {
    Uuid::from_bytes([n; 16])
}

/// Build a Grundschule-shaped `Problem`. Mirrors the test fixture in
/// `solver-core/tests/grundschule_smoke.rs::grundschule()`. Asserts the
/// resulting problem has exactly 40 lessons so copy-paste drift is caught.
fn grundschule_fixture() -> Problem {
    let time_blocks: Vec<TimeBlock> = (0..25)
        .map(|i| TimeBlock {
            id: TimeBlockId(bench_uuid(100 + i)),
            day_of_week: i / 5,
            position: i % 5,
        })
        .collect();

    let teachers: Vec<Teacher> = (0..8)
        .map(|i| Teacher {
            id: TeacherId(bench_uuid(30 + i)),
            max_hours_per_week: 28,
        })
        .collect();

    let rooms: Vec<Room> = (0..5)
        .map(|i| Room {
            id: RoomId(bench_uuid(50 + i)),
        })
        .collect();

    let subject_ids: Vec<SubjectId> = (0..8).map(|i| SubjectId(bench_uuid(60 + i))).collect();
    let subjects: Vec<Subject> = subject_ids.iter().map(|id| Subject { id: *id }).collect();

    let classes: Vec<SchoolClass> = (0..2)
        .map(|i| SchoolClass {
            id: SchoolClassId(bench_uuid(70 + i)),
        })
        .collect();

    let hours_per_class: [[u8; 8]; 2] = [
        [6, 5, 2, 0, 2, 1, 2, 3],
        [5, 5, 4, 2, 2, 1, 2, 3],
    ];

    let mut lessons = Vec::new();
    let mut quals = Vec::new();
    let mut lesson_idx: u8 = 0;
    for (c_idx, class) in classes.iter().enumerate() {
        for (s_idx, subject) in subjects.iter().enumerate() {
            let hours = hours_per_class[c_idx][s_idx];
            if hours == 0 {
                continue;
            }
            let teacher = &teachers[(c_idx * 4 + s_idx) % teachers.len()];
            lessons.push(Lesson {
                id: LessonId(bench_uuid(200 + lesson_idx)),
                school_class_id: class.id,
                subject_id: subject.id,
                teacher_id: teacher.id,
                hours_per_week: hours,
            });
            lesson_idx += 1;
            quals.push(TeacherQualification {
                teacher_id: teacher.id,
                subject_id: subject.id,
            });
        }
    }

    // Total hours across both classes: 21 + 24 = 45 individual (class, subject) lessons
    // after filtering zero-hour entries. We expect exactly 15 lessons (8 non-zero for
    // class 1/2 minus the one zero-hour Fremdsprache = 7, plus 8 for class 3/4 = 15).
    assert_eq!(lessons.len(), 15, "bench fixture drifted from the test fixture");

    // Gym (room index 4) suits only Sport (subject index 7).
    let suits: Vec<RoomSubjectSuitability> = vec![RoomSubjectSuitability {
        room_id: rooms[4].id,
        subject_id: subject_ids[7],
    }];

    Problem {
        time_blocks,
        teachers,
        rooms,
        subjects,
        school_classes: classes,
        lessons,
        teacher_qualifications: quals,
        teacher_blocked_times: vec![],
        room_blocked_times: vec![],
        room_subject_suitabilities: suits,
    }
}

fn bench_grundschule(c: &mut Criterion) {
    let problem = grundschule_fixture();

    let expected_hours: u32 = problem
        .lessons
        .iter()
        .map(|l| u32::from(l.hours_per_week))
        .sum();

    let mut group = c.benchmark_group("solver");
    group.sample_size(BENCH_SAMPLE_COUNT);
    group.sampling_mode(SamplingMode::Flat);

    let mut per_iter: Vec<Duration> = Vec::with_capacity(BENCH_SAMPLE_COUNT * 4);
    group.bench_function("grundschule", |b| {
        b.iter_custom(|iters| {
            let mut total = Duration::ZERO;
            for _ in 0..iters {
                let start = Instant::now();
                let solution = solve(&problem).expect("solve must succeed on the grundschule fixture");
                let elapsed = start.elapsed();
                total += elapsed;
                per_iter.push(elapsed);

                // Sanity: we should not be benching a degenerate placement.
                assert!(solution.violations.is_empty());
                assert_eq!(solution.placements.len() as u32, expected_hours);
                // Room no-double-book invariant, cheap to verify and cheaper than a silent
                // regression on a future algorithm change.
                let mut seen: HashSet<(RoomId, TimeBlockId)> = HashSet::new();
                for pl in &solution.placements {
                    assert!(seen.insert((pl.room_id, pl.time_block_id)));
                }
            }
            total
        });
    });
    group.finish();

    // Compute and emit the fenced output block. `per_iter` contains every
    // sample criterion recorded; `iter_custom` is called potentially many
    // times but each call appends one or more entries.
    let total_samples = per_iter.len();
    let (p1, p50, p99) = compute_percentiles(&mut per_iter);
    let mean = per_iter.iter().copied().sum::<Duration>() / total_samples as u32;
    let placements_per_sec = (f64::from(expected_hours) / mean.as_secs_f64()) as u64;

    eprintln!("---SOLVER-BENCH-BASELINE---");
    eprintln!("fixture\tgrundschule");
    eprintln!("samples\t{}", total_samples);
    eprintln!("p1_us\t{}", p1.as_micros());
    eprintln!("p50_us\t{}", p50.as_micros());
    eprintln!("p99_us\t{}", p99.as_micros());
    eprintln!("placements_per_sec\t{}", placements_per_sec);
    eprintln!("total_placements\t{}", expected_hours);
    eprintln!("total_hard_violations\t0");
    eprintln!("---END---");
}
```

```bash
#!/usr/bin/env bash
# Regenerate solver/solver-core/benches/BASELINE.md.
#
# Runs the grundschule criterion bench, parses the fenced
# ---SOLVER-BENCH-BASELINE--- block from stderr, and renders the committed
# baseline markdown with a host-info footer. Idempotent: two runs on the
# same host overwrite the same file; only the numbers change.

set -euo pipefail

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
OUT="${ROOT}/solver/solver-core/benches/BASELINE.md"
STDERR_FILE="$(mktemp)"
trap 'rm -f "${STDERR_FILE}"' EXIT

cd "${ROOT}"
cargo bench -p solver-core --bench solver_grundschule 2> "${STDERR_FILE}"

# Extract the fenced block. awk emits only the lines between the markers,
# exclusive. If the block is missing the output is empty and we error below.
BLOCK="$(awk '/^---SOLVER-BENCH-BASELINE---$/{flag=1; next} /^---END---$/{flag=0} flag' "${STDERR_FILE}")"
if [[ -z "${BLOCK}" ]]; then
    echo "error: SOLVER-BENCH-BASELINE block not found in bench stderr" >&2
    cat "${STDERR_FILE}" >&2
    exit 1
fi

get() {
    local key="$1"
    local line
    line="$(awk -v k="${key}" -F'\t' '$1 == k {print $2}' <<< "${BLOCK}")"
    if [[ -z "${line}" ]]; then
        echo "error: key ${key} missing from bench output" >&2
        exit 1
    fi
    printf '%s' "${line}"
}

FIXTURE="$(get fixture)"
SAMPLES="$(get samples)"
P1="$(get p1_us)"
P50="$(get p50_us)"
P99="$(get p99_us)"
PPS="$(get placements_per_sec)"
PLACEMENTS="$(get total_placements)"
HARD="$(get total_hard_violations)"

CPU="$(awk -F': ' '/^model name/{print $2; exit}' /proc/cpuinfo || echo unknown)"
KERNEL="$(uname -r)"
RUSTC="$(rustc --version)"
DATE="$(date -Idate)"

cat > "${OUT}" <<EOF
# Solver bench baseline

<!-- Regenerated by \`mise run bench:record\`. Do not hand-edit. -->

| Fixture | Samples | p1 (µs) | p50 (µs) | p99 (µs) | Placements/sec | Placements | Hard violations |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| ${FIXTURE} | ${SAMPLES} | ${P1} | ${P50} | ${P99} | ${PPS} | ${PLACEMENTS} | ${HARD} |

Refreshed ${DATE} on ${CPU}, Linux ${KERNEL}, ${RUSTC}.

Refresh with \`mise run bench:record\` when a PR intentionally changes solver
performance. An accidental diff here is a review gate, not a feature.

See \`solver/CLAUDE.md\` (Bench workflow) for the 20% regression budget and the
algorithm-phase PRs this baseline gates.
EOF

echo "wrote ${OUT}"
```

- **Fixture:** today one Grundschule-shaped fixture (2 classes, 8 teachers, 5 rooms, 15 lessons, 45 placements). Sprint item 6 on `OPEN_THINGS.md` adds zweizuegig and Gesamtschule fixtures.
