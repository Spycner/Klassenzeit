#### Hessen Grundschule reference data

Researched 2026-04-22 and mapped onto the schema during the seed brainstorm (`docs/superpowers/specs/2026-04-24-grundschule-seed-design.md` "Research notes" table). Values below are the source figures; the seed reflects every "Yes" row and documents every "Not encoded" row against an existing OPEN_THINGS deferral.

- *Stundentafel Klasse 1/2:* 21 Pflichtstunden (Deutsch 6, Mathematik 5, Sachunterricht 2, Religion/Ethik 2, Kunst/Werken/Musik 3, Sport 3) plus 2 Stunden Förderunterricht/AGs.
- *Stundentafel Klasse 3/4:* 25 Pflichtstunden (Deutsch 5, Mathematik 5, Sachunterricht 4, Fremdsprache 2, Religion/Ethik 2, Kunst/Werken/Musik 4, Sport 3) plus 2 Stunden Förderunterricht/AGs. Über alle vier Jahrgänge also 92 Wochenstunden gesamt.
- *Lehrer-Pflichtstunden (Grundschule):* 28 Wochenstunden Vollzeit, ab 01.02.2026 reduziert auf 27,5. Teilzeit wird anteilig als Bruch geführt (typische Werte: 14/28, 18/28, 21/28). Lebensarbeitszeitkonto: 0,5 Stunden pro Woche Gutschrift bis zum 60. Lebensjahr (anteilig bei Teilzeit).
- *WeekScheme-Zeitraster:* Unterrichtsbeginn 7:45 bis 8:15, Unterrichtsstunde = 45 Minuten. Zwei Hofpausen (je 15 bis 20 Minuten, nach der 2. und nach der 4. Stunde) plus eine kurze Frühstückspause im Klassenraum (ca. 10 Minuten). Tagesende 11:30 bis 13:20 im Halbtag, bis 14:55 im Ganztag. Ganztagsschulen ergänzen eine Mittagspause von 45 bis 60 Minuten. The shipped seed uses a 7-period grid (08:00 to 14:05, Periode 7 ab 13:20) to give the MVP greedy solver enough slack; revisit to a 6-period Halbtag once FFD or LAHC ship (PRs 7 and 9 in the active sprint).
- *Quellen:* Hessisches Kultusministerium, Hessischer Bildungsserver, GEW Hessen Pflichtstundenverordnung.

## Acknowledged deferrals

Items deliberately out of scope of any active sprint, with rationale for when to revisit. Sprint PRs may touch them in passing; do not retrofit out-of-scope changes into a sprint PR.

- **Repository / unit-of-work layer.** Routes currently take `AsyncSession` directly. The schedule handler loads 7 to 8 entity types in one call and duplicates some of the existing CRUD queries. The earlier guidance ("add when it hurts") still applies; if a handler grows past ~80 lines, file the pain as a follow-up rather than detouring mid-sprint.
- **`active` flag on WeekScheme.** Matters only if a school has multiple week schemes with one "live" scheme. For the prototype, a class points at one `week_scheme_id` directly, which is enough. Revisit when the readiness checklist or a schedule-switcher needs it.
- **Auto-infer WeekScheme time-block position + validate ordering.** UX polish that does not affect the solver or the schedule view. Today the time-blocks form asks users to type a period number, and nothing prevents overlapping or out-of-order periods on the same day. Change to: drop the `position` input; backend assigns `position` as the chronological rank (by `start_time`) among blocks on the same day, renumbering siblings in one transaction on insert/edit; validate that consecutive pairs on the same day sorted by `start_time` satisfy `start_time >= previous end_time`, returning 422 on overlap. Frontend mirrors the check and maps 422 to a form root error, following the existing 409-duplicate pattern. DB column and `(day, position)` uniqueness stay as-is. Reported during frontend small-fixups session.
- **Whole-school cross-class consistency.** `POST /api/classes/{id}/schedule` writes only placements for the requested class. Sibling classes' persisted placements are not fed back into the solver's input, so a per-class solve can legitimately produce placements that overlap a persisted sibling's `(time_block, room)` pair. Add persisted-schedule ingestion into `build_problem_json` when a "whole-school generate" view lands, or when demos surface the overlap as user-visible breakage. Reported during placement-persistence work.
- **Persist solver violations or surface them on GET.** Violations are returned by `POST /api/classes/{id}/schedule` but not persisted. `GET /api/classes/{id}/schedule` returns placements only. If the schedule view needs a persistent "why is this incomplete?" diagnostic, either add a `schedule_violations` table or teach the GET endpoint to re-run the solver and surface fresh violations on demand. Reported during placement-persistence work.
- **Advisory lock on concurrent POSTs for the same class.** Current guard is Postgres row-level locks inside the delete-then-insert transaction (last writer wins). An explicit `pg_advisory_xact_lock(hashtext(class_id::text))` would prevent wasted parallel solves when demo users click Generate twice. Add if traffic shows interleaving. Reported during placement-persistence work.
- **Parallel lesson groups (Religion / Ethik ev/kath/Ethik).** Real Hessen Grundschulen run Religion ev, Religion kath, and Ethik in parallel time-blocks so every child attends exactly one; they share the slot but occupy different rooms with different teachers. The schema has one `Lesson` per `(school_class, subject)` pair and one `teacher_id` per lesson, so the parallelisation is not representable. Adding a `lesson_group_id` column (nullable) on `lessons` plus a rule "lessons sharing a group must share the time_block but must hold different `(teacher_id, room_id)` pairs" is the smallest schema change; solver would need a corresponding hard constraint. Defer until a customer school actually runs parallel Religion groups and we have a concrete UX in mind. Surfaced during grundschule seed research.
- **SchoolClass headcount / Klassenobergrenze.** `SchoolClass` has no `headcount` column, so the Hessen Klassenobergrenze (22-25 for grades 1/2, 25-28 for grades 3/4) and Room `capacity` cannot be cross-checked. Adding `headcount: int | None` on `SchoolClass` plus a pre-solve validator "all rooms a lesson can land in must have `capacity >= school_class.headcount` (or null)" is cheap; landing the pre-solve check is a separate item. Not a demo blocker since the seeded Klassenräume all use capacity 25. Surfaced during grundschule seed research.
- **Schwimmunterricht: external location + travel time.** Hessen Klasse 3 has one Doppelstunde Schwimmen per week at an auswärtige Schwimmhalle, with 10 to 15 minutes of Wegezeit each way that blocks the preceding and following time-blocks for the class and the teacher. The schema has no concept of "external room" or "pre-block / post-block buffer". Prerequisites: Doppelstunden support (sprint PR 8) plus a new `Room.is_external` flag and per-lesson buffer metadata. Out of scope until a customer school wants Schwimmen modelled. Surfaced during grundschule seed research.
- **Doppelbesetzung / Förderstunden with two teachers.** In Klasse 1/2, Förderunterricht often runs as a co-taught regular lesson (one Klassenlehrer + one Förderlehrkraft in the same slot). `Lesson` has exactly one `teacher_id`, so the second teacher cannot be attached. Smallest change: a `Lesson.co_teacher_id: uuid.UUID | None` column plus solver-side "if set, co-teacher must also be free and qualified in the lesson's subject". Alternatively, model as a second parallel Lesson tied via `lesson_group_id` (see Religion-parallel item). Defer until real Förder teachers exist in the data. Surfaced during grundschule seed research.
- **Pausen- / Aufsichtspflichten and Vertretungsreserve.** Teachers owe duty-minutes during Hofpausen on a rota; substitution reserve reduces their teaching capacity below `max_hours_per_week`. The schema has no break metadata on TimeBlocks (breaks are implicit gaps) and no reserve budget on Teacher. Two separate follow-ups: (a) `TimeBlock.kind` enum or a sibling `Break` table so supervision can be scheduled; (b) `Teacher.reserve_hours_per_week` with the solver subtracting it from the max. Out of scope until a customer school tracks supervision. Surfaced during grundschule seed research.
- **Teilzeit-Tage patterns on Teacher.** Part-time teachers are often contracted to specific weekdays (e.g. Dana works Mo, Di, Mi). Today this is representable via `teacher_availabilities` rows marking all time-blocks on off-days as unavailable, but that is verbose (15 rows per off-day) and the seed leaves availability empty. A `Teacher.working_days: set[int]` column or a higher-level "weekly pattern" model would let users describe "half-time, Mo-Mi" in one field. Revisit when a second real Teilzeit teacher enters demo data and the verbose availability grid starts to feel painful. Surfaced during grundschule seed research.
- **Lebensarbeitszeitkonto (teacher seniority time credit).** Hessen GEW Pflichtstundenverordnung: teachers earn 0.5h/week into a long-service time account until age 60, anteilig für Teilzeit. Not representable today (no age or seniority metadata on Teacher; no time-account model). Low priority since the credit is a payroll concept, not a scheduling hard constraint; document the deferral so future users asking "why isn't 27.5 in the seed?" see the answer. Surfaced during grundschule seed research.
- **Klassenraum-per-class (home room) mapping.** Hessen Grundschulen operate under a Klassenraum-Prinzip: every class has a home room where most non-specialty lessons happen. Today `School` / `SchoolClass` has no "home room" FK, so the solver picks whichever suitable Klassenraum is free for each (class, time_block) pair. Seed rooms are named `Klasse 1a` / `Klasse 2a` / ... so the demo looks right only by coincidence when the solver happens to route 1a into the matching Klassenraum. Smallest change: `SchoolClass.home_room_id: uuid.UUID | None` plus a soft-preference "prefer home_room_id when suitable" inside the scoring function (sprint PR 9's soft-constraint framework makes this cheap to add afterward). Surfaced during grundschule seed research.
- **`demo_gesamtschule` bench fixture (sprint item 6, gesamtschule leg).** 24 classes (Sek I, grades 5-10, four Züge), ~50 teachers, ~31 rooms, ~700 placements; pre-assigned `teacher_id` per Lesson; matched Rust mirror in `solver/solver-core/benches/solver_fixtures.rs`. Spec at [`2026-04-26-bench-fixture-matrix-design.md`](specs/2026-04-26-bench-fixture-matrix-design.md) and plan at [`2026-04-26-bench-fixture-matrix.md`](plans/2026-04-26-bench-fixture-matrix.md) keep the deferred Tasks 5 + 6 verbatim under `DEFERRED` markers; the follow-up PR can pick them up unchanged. Deferred from the zweizügige PR because the seed authoring (50-teacher pool, 240-entry `_TEACHER_ASSIGNMENTS_GESAMTSCHULE` dict) and the risk that the MVP greedy solver cannot solve cleanly without seed iteration made the cost asymmetric for an autonomous PR. Revisit after items 7 to 9 (FFD / LAHC) ship and the third size's measurement value is concrete.
- **Synthetic bench fixture generator (sprint item 6, optional leg).** A parametric `gen_synthetic_problem(num_classes, num_teachers, num_subjects, num_rooms)` producing a deterministic `Problem` JSON for stress tests at 100+ classes. Originally optional in the OPEN_THINGS item 6 description; deferred from the zweizügige PR. Revisit when items 7 to 9 surface a concrete need that the named-fixture matrix (grundschule + zweizuegig + gesamtschule once that lands) cannot answer.

### Testing

- **Session-scoped event loop may cause timing interference at scale.** The `asyncio_default_fixture_loop_scope = "session"` setting (introduced for the Chunk G DB fixtures) means all async tests and fixtures share a single event loop for the entire pytest session. This prevents asyncpg "Future attached to a different loop" errors with session-scoped fixtures but means one slow or stalled async test can delay all subsequent tests in the session. Not a problem with the current 16-test suite, but worth revisiting if the test suite grows large or if tests with long async timeouts are added.

  1. **Pixel-diff snapshots** (Playwright `toHaveScreenshot`, Percy, Chromatic). Catches any pixel change, noisy on intentional design tweaks. Fallback if design stabilises enough to make a baseline meaningful.
  3. **Vision-LLM diff.** Crop before/after screenshots of the same element, pass both to a VLM with "do these shapes match, ignoring color and text". Fuzzier, catches emergent layout differences that don't trace to a single CSS property (pseudo-element sizing, child layout shift). Use as a follow-up when the computed-style diff says styles match but something still looks off.

- **Shell-exported `KZ_ENV=dev` defeats pytest router mounting.** The `os.environ.setdefault` in conftest no-ops if the shell already has `KZ_ENV` set. A shell-exported `KZ_ENV=dev` would silently skip mounting the testing router, and router tests would fail with 404. Add a warning in conftest or switch to `pytest-env` if this bites anyone.
