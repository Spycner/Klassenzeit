# Step 5b: i18n (DE/EN) — Design Spec

## Goal

Add bilingual support (German/English) to the Klassenzeit frontend using `next-intl`, matching the pattern used in the pascalkraus.com website (`/home/pascal/Code/website/`).

## Scope

- Install and configure `next-intl`
- Restructure routes under `[locale]` segment
- Extract all hardcoded UI strings into translation files
- Add language switcher component
- German as default locale

Out of scope: backend i18n, date/number formatting beyond what next-intl provides by default, RTL support.

## Architecture

### Configuration

Three new files:

**`frontend/i18n/config.ts`**
```ts
export const locales = ["de", "en"] as const;
export const defaultLocale = "de" as const;
export type Locale = (typeof locales)[number];
```

**`frontend/i18n/request.ts`**
```ts
import { getRequestConfig } from "next-intl/server";
import { defaultLocale, locales } from "@/i18n/config";

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;
  if (!locale || !(locales as readonly string[]).includes(locale)) {
    locale = defaultLocale;
  }
  return {
    locale,
    messages: (await import(`@/messages/${locale}.json`)).default,
  };
});
```

**`frontend/next.config.ts`** — wrap existing config with `createNextIntlPlugin("./i18n/request.ts")`.

### Route Structure

Move all pages under `src/app/[locale]/`:

```
src/app/
├── layout.tsx                          # Minimal: just <html><body>
├── page.tsx                            # Redirect to /${defaultLocale}/schools
├── [locale]/
│   ├── layout.tsx                      # Providers + NextIntlClientProvider + locale validation
│   ├── schools/
│   │   ├── page.tsx                    # Schools list
│   │   └── [id]/
│   │       ├── layout.tsx              # School sidebar layout
│   │       ├── page.tsx                # School dashboard
│   │       └── members/
│   │           └── page.tsx            # Members page
```

**Root layout** (`src/app/layout.tsx`): Returns `children` directly — no `<html>` or `<body>` tags. Imports `globals.css`. Following the same pattern as the reference website.

**Locale layout** (`src/app/[locale]/layout.tsx`): Renders `<html lang={locale}>` and `<body>` with fonts. Validates locale (calls `notFound()` if invalid), loads messages via `getMessages()`, wraps children with `NextIntlClientProvider`, then existing providers (ThemeProvider, TooltipProvider, SchoolProvider, KeycloakProvider).

**Root page** (`src/app/page.tsx`): Redirects to `/${defaultLocale}/schools`.

### Translation Files

| Namespace | Used by |
|-----------|---------|
| `common` | Shared strings: loading, save, cancel, edit, remove, create, logout |
| `meta` | Page title and description |
| `auth` | KeycloakProvider loading/error states |
| `schools` | Schools list page and create dialog |
| `school` | School dashboard, sidebar nav, edit dialog |

### Language Switcher

New component: `src/components/language-switcher.tsx`

- Client component using `useLocale()`, `usePathname()`, `useRouter()` from next-intl/next
- Replaces locale segment in current path and navigates via `router.push()`
- Renders a ghost button showing the target locale (e.g., "EN" when viewing German)

**Placement:**
- In the school sidebar layout, next to the Logout button
- On the schools list page, in a small toolbar at the top-right

### Component Changes

All pages and components switch from hardcoded strings to `useTranslations("namespace")`:

| Component | Namespace(s) |
|-----------|-------------|
| `src/app/[locale]/layout.tsx` | `meta` (for metadata) |
| `src/app/[locale]/schools/page.tsx` | `schools`, `common` |
| `src/app/[locale]/schools/[id]/layout.tsx` | `school`, `common` |
| `src/app/[locale]/schools/[id]/page.tsx` | `school`, `common` |
| `src/app/[locale]/schools/[id]/members/page.tsx` | `members`, `common` |
| `src/components/providers/keycloak-provider.tsx` | `auth` |

All internal `<Link>` hrefs and `router.push()` calls include `/${locale}` prefix. The locale is obtained via `useLocale()` hook.

### No Other Changes

Layout, styling, component structure, and behavior remain unchanged. The only visible addition is the language toggle button.

## Testing

- Verify all pages render correctly in both `/de/...` and `/en/...` routes
- Verify language switcher navigates to the other locale while preserving the current path
- Verify invalid locale paths return 404
- Verify root `/` redirects to `/de/schools`
- Verify all strings are translated (no hardcoded text remaining in components)
