# Realistic Hessen Grundschule demo seed

**Date:** 2026-04-24
**Status:** Design approved, plan pending.

## Problem

The prototype sprint (`docs/superpowers/OPEN_THINGS.md`) has shipped the solver MVP, placement persistence, and the frontend schedule view (PR #124). The remaining sprint step 1 is a demo seed: "a one-shot command that creates the week scheme, Stundentafeln for grades 1 to 4, plausible teachers / rooms, and a pair of classes ready to generate lessons + schedule." Without this seed there is no realistic data to demo the end-to-end flow and no fixture to feed the Playwright smoke (sprint step 2).

The seed also pays off an explicit research instruction on `OPEN_THINGS.md:25`: the Hessen Stundentafel numbers captured in the sprint section were a first pass; additional constraints (Klassenlehrer-Prinzip, Raumtypen und Eignung, Fachlehrer-Einsatz, Pflichtstunden-Regeln) needed to be weighed and mapped onto the current schema before seeding.

## Goal

One backend PR that:

1. Adds a new module `klassenzeit_backend.seed.demo_grundschule` holding the data and an `async def seed_demo_grundschule(session: AsyncSession) -> None` coroutine.
2. Adds a Typer subcommand `seed-grundschule` to the existing `klassenzeit-backend` CLI that wraps the coroutine in a session + single commit, refusing to run when `settings.env == "prod"`.
3. Produces realistic Hessen Grundschule data grounded in the reference figures captured at `OPEN_THINGS.md` lines 20-25 and the mapping table in this spec.
4. Ships three focused tests: shape, fail-fast rollback, and solvability (seed â†’ generate lessons â†’ solver â†’ zero hard violations).

After this PR, `uv run klassenzeit-backend seed-grundschule` creates everything the frontend needs for the user to click "Generate lessons" and "Generate schedule" and see a feasible, pedagogically sensible-looking timetable.

## Non-goals

- **The Playwright E2E smoke test** that drives the seed end-to-end. Sprint step 2, separate PR.
- **A `/__test__/seed-grundschule` endpoint.** The test-only router change belongs with the Playwright PR that consumes it.
- **Lessons.** The seed stops before lesson rows; lessons are generated by the existing "Generate lessons" action in the UI/backend.
- **Doppelstunden / `preferred_block_size > 1`.** The solver rejects it today; noted under OPEN_THINGS "Doppelstunden support". All seeded lessons will use `preferred_block_size = 1`.
- **Schwimmunterricht for Klasse 3.** Requires Doppelstunden + an auswÃ¤rtige Halle concept the schema lacks. Deferred alongside Doppelstunden.
- **Parallel Religion/Ethik lessons.** No schema support for "parallel lesson groups"; the seed collapses ev/kath/Ethik into one `Religion/Ethik` subject.
- **Doppelbesetzung / FÃ¶rderstunden in Klasse 1/2.** Lesson has one `teacher_id`; double-staffing is not representable today.
- **Soft constraints** (HauptfÃ¤cher frÃ¼h, Sport nicht nach Essen, Klassenlehrer-Gap-Minimierung). Covered by the OPEN_THINGS "Soft constraints with an objective function" backlog item.
- **Klassenraum per class.** No "home room" field; the solver picks whichever suitable room is free.
- **A second seed** (Gymnasium, Gesamtschule). YAGNI; single file, extract helpers if a second seed arrives.
- **An ADR.** No load-bearing architectural decision is introduced. The seed is demo data that uses existing entities, routes, and the existing solver.

### Data shape

#### WeekScheme and TimeBlocks

One WeekScheme: name `Grundschule Zeitraster`, description `Hessen Grundschule Halbtag: 5 Tage, 6 Stunden a 45 Minuten, Hofpausen nach der 2. und 4. Stunde`.

30 TimeBlocks on a Monday-to-Friday grid with 6 periods each. Period times reflect a typical Hessen Grundschule Halbtag:

| Period | Start | End |
|---|---|---|
| 1 | 08:00 | 08:45 |
| 2 | 08:45 | 09:30 |
| 3 | 09:50 | 10:35 |
| 4 | 10:35 | 11:20 |
| 5 | 11:35 | 12:20 |
| 6 | 12:20 | 13:05 |

Hofpausen 09:30 to 09:50 (after period 2) and 11:20 to 11:35 (after period 4) are implicit gaps, not TimeBlocks. The schema has no "break" concept; the solver only cares about when teaching may happen.

`day_of_week` uses the repo convention confirmed by `backend/src/klassenzeit_backend/scheduling/schemas/week_scheme.py` (`Field(ge=0, le=4)`) and `frontend/src/i18n/day-keys.ts`: 0 = Monday through 4 = Friday. `position` is the period index, 1 through 6.

### CLI wiring

One new Typer command alongside the existing three in `backend/src/klassenzeit_backend/cli.py`:

```python
@cli.command()
def seed_grundschule() -> None:
    """Seed a demo Hessen Grundschule (4 classes, 6 teachers, 7 rooms)."""
    settings = get_settings()
    if settings.env == "prod":
        typer.echo("seed-grundschule is disabled in production", err=True)
        raise typer.Exit(code=1)
    try:
        asyncio.run(_run_seed_grundschule())
    except IntegrityError as exc:
        typer.echo(
            f"Seed aborted (integrity error): {exc.orig}\n"
            "The database already contains conflicting rows. "
            "Reset with `mise run db:reset` (dev) or the /__test__/reset endpoint (test) and try again.",
            err=True,
        )
        raise typer.Exit(code=1) from exc
    typer.echo("Grundschule demo seeded successfully.")


async def _run_seed_grundschule() -> None:
    settings = get_settings()
    engine = create_async_engine(str(settings.database_url))
    factory = async_sessionmaker(engine, expire_on_commit=False)
    try:
        async with factory() as session:
            await seed_demo_grundschule(session)
            await session.commit()
    finally:
        await engine.dispose()
```

Typer subcommand name auto-resolves from the Python function name (`seed_grundschule` â†’ `seed-grundschule`), matching the repo's existing CLI conventions.

### Seed coroutine shape

`seed_demo_grundschule(session: AsyncSession)` inserts in FK-topological order with `await session.flush()` at each layer so primary-key UUIDs are available for the next wave:

1. WeekScheme, flush, then 30 TimeBlocks referencing it, flush.
2. All 9 Subjects, flush.
3. Four Stundentafeln, flush, then 40 StundentafelEntry rows, flush.
4. Four SchoolClasses referencing Stundentafeln + WeekScheme, flush.
5. Six Teachers, flush, then TeacherQualification rows (M:N into Subjects) for the matrix above, flush.
6. Seven Rooms, flush, then 27 RoomSubjectSuitability rows, flush.

No `session.commit()` inside the coroutine. The caller commits once at the end; any exception anywhere rolls back the entire transaction.

Helpers named with the `demo_grundschule_` prefix to satisfy the "unique function names globally" rule: `demo_grundschule_time_blocks`, `demo_grundschule_subjects`, `demo_grundschule_stundentafel_entries`, `demo_grundschule_teacher_qualifications`, `demo_grundschule_room_suitabilities`. Each helper returns pure tuples / dicts so shape tests can call them in isolation without a DB.

### Tests

Three test files under `backend/tests/seed/`:

- `backend/tests/seed/__init__.py`: empty.
- `backend/tests/seed/test_demo_grundschule_shape.py`: asserts entity counts (9 subjects, 4 Stundentafeln, 34 StundentafelEntries, 4 SchoolClasses, 6 Teachers, 22 TeacherQualifications, 7 Rooms, 27 RoomSubjectSuitabilities, 1 WeekScheme, 35 TimeBlocks), FK integrity (each SchoolClass's Stundentafel has the same `grade_level`; every StundentafelEntry's `subject_id` exists; every TeacherQualification references an existing teacher + subject), and per-tafel hour sums (23 for grades 1-2, 26 for grades 3-4).
- `backend/tests/seed/test_demo_grundschule_rollback.py`: pre-inserts a Subject named `Deutsch`, calls `seed_demo_grundschule(session)` inside a nested transaction, expects `IntegrityError`, then asserts `select count(*)` for Subject stays at 1 (the pre-existing row).
- `backend/tests/seed/test_demo_grundschule_solvability.py`: calls `seed_demo_grundschule(session)` + `session.commit()`, then for each SchoolClass invokes the existing generate-lessons helper that the API route uses (located in `scheduling/routes/lessons.py` or similar; exact import resolved during plan), then invokes the existing solve helper (also from `scheduling/routes`), asserts that every solve returns zero `violations`. Marked with a comment noting it is the slowest seed test; no xfail or skip.

`backend/tests/seed/conftest.py` only added if a shared fixture emerges; expected to not exist on first landing.

## Risks

- **Becker / Hoffmann capacity pinch.** Becker has 18h to cover 8h RE + 4h MU + some FÃ– = 12h+. Hoffmann has 21h to cover 12h SP + some KU + some FÃ–. KU total is 8h (2h Ã— 4 classes) and MÃ¼ller/Schmidt split the primary grades' KU with Hoffmann covering none or a subset of grades 3-4; FÃ– total is 8h. Becker residual 6h, Hoffmann residual 9h after SP. Both capacities comfortably cover demand with slack. The solvability test is the canary.
- **Schema drift.** If `teacher_qualifications` grows a new column or `stundentafel_entries` gains a `preferred_block_size` server default change before this PR lands, seed constants may fail to insert. Mitigation: plan step `mise run db:reset` + `mise run db:migrate` dry run before committing feat.
- **Day-of-week convention drift.** Backend schema pins `day_of_week: int = Field(ge=0, le=4)` (Monday = 0), frontend `dayShortKey` / `dayLongKey` consume the same 0 to 4 range. The seed's TimeBlocks use 0 through 4 accordingly. If any helper in the plan accidentally emits 1 to 5, the schema's `Field(ge=0, le=4)` will fail on insert and the plan's shape tests will catch it.
- **Subject name collisions with existing dev data.** If a developer already has a Subject named `Deutsch` in their local DB, seed aborts cleanly (Q4 fail-fast). Rollback test locks in this behaviour.
