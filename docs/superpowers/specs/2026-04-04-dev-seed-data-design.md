# Dev Seed Data — Design Spec

## Goal

Provide realistic German primary school data so `just dev` gives developers a fully populated app without manual setup.

## Problem

After `docker compose up` and migrations, all tables are empty. The 3 Keycloak test users exist but have no `app_users` rows, no `school_memberships`, and no reference data. The app is unusable until data is manually created through the UI or API.

## Approach: Two-Part Seeding

### Part 1: SQL Seed File (`docker/seeds/dev-seed.sql`)

A plain SQL script that populates the school and all reference data. Mounted as a Postgres init script so it runs automatically on first `docker compose up`.

**Tables seeded:**

| Table | Count | Details |
|-------|-------|---------|
| `schools` | 1 | "Grundschule am See", slug `grundschule-am-see` |
| `school_years` | 1 | 2025/2026, is_current=true |
| `terms` | 2 | 1. Halbjahr (current, Sep-Jan), 2. Halbjahr (Feb-Jul) |
| `teachers` | 8 | German names, 2 part-time, realistic hours |
| `subjects` | 8 | Deutsch, Mathematik, Englisch, Sachunterricht, Sport, Musik, Kunst, Religion |
| `rooms` | 6 | 4 classrooms (101-104), Turnhalle, Musikraum |
| `school_classes` | 4 | 1a, 2a, 3a, 4a (grades 1-4) |
| `time_slots` | 30 | 6 periods/day × 5 days, standard German primary school schedule |
| `teacher_subject_qualifications` | ~16 | Each teacher qualified for 2 subjects |
| `teacher_availabilities` | ~4 | Part-time teachers blocked on certain days |
| `room_subject_suitabilities` | 2 | Turnhalle→Sport, Musikraum→Musik |
| `curriculum_entries` | ~20 | Realistic hours for 1. Halbjahr, 4 classes |

**Not seeded in SQL:**
- `app_users` — depends on Keycloak-generated UUIDs
- `school_memberships` — depends on `app_users`
- `lessons` — generated by the scheduler

**Properties:**
- Idempotent: `INSERT ... ON CONFLICT DO NOTHING`
- Uses well-known UUIDs (all based on `00000000-0000-0000-0000-*`) for easy cross-referencing
- All timestamps use `NOW()` function

### Part 2: Bootstrap Script (`docker/seeds/bootstrap.sh`)

A shell script that creates `app_users` and `school_memberships` by fetching real Keycloak user IDs.

**Flow:**
1. Wait for Keycloak to be ready (health check loop)
2. Authenticate as Keycloak admin (`admin`/`admin`)
3. Fetch user list from `klassenzeit` realm
4. For each test user (admin@test.com, teacher@test.com, viewer@test.com):
   - Extract the Keycloak-generated UUID
   - Insert `app_users` row with correct `keycloak_id`
   - Insert `school_memberships` row with appropriate role
5. All inserts use `ON CONFLICT DO NOTHING`

## Integration

### Docker Compose Changes

Mount `dev-seed.sql` as a Postgres init script:

```yaml
postgres-dev:
  volumes:
    - postgres_dev_data:/var/lib/postgresql/data
    - ./docker/seeds/dev-seed.sql:/docker-entrypoint-initdb.d/02-seed.sql:ro
```

The `02-` prefix ensures it runs after any `01-` schema init. However, since we use SeaORM migrations (not init scripts) for schema, the seed SQL must run after migrations. This means the init script approach won't work directly — tables don't exist yet when postgres starts.

**Revised approach:** The seed SQL runs via `just db-seed`, not as a postgres init script. The justfile `dev` recipe will be updated to run migrations + seed in sequence.

### Justfile Recipes

```just
# Seed dev database with sample data
db-seed:
    docker exec klassenzeit-postgres-dev psql -U postgres -d klassenzeit_dev -f /seeds/dev-seed.sql

# Bootstrap user accounts from Keycloak
db-bootstrap:
    bash docker/seeds/bootstrap.sh

# Full dev setup: start containers, migrate, seed, bootstrap
dev-setup:
    just dev
    @echo "Waiting for services..."
    sleep 5
    just db-migrate
    just db-seed
    just db-bootstrap
```

Mount the seeds directory in docker-compose:

```yaml
postgres-dev:
  volumes:
    - postgres_dev_data:/var/lib/postgresql/data
    - ./docker/seeds:/seeds:ro
```

- Musik: 2h
- Kunst: 2h
- Religion: 2h

## Out of Scope

- Multiple schools
- Lessons (scheduler generates these)
- Teacher availability overrides per term
- Complex week patterns (A/B weeks)
